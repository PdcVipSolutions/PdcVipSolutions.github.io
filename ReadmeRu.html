WorkSpace Manager

Документация  WorksSpace Manager помещена в SpbRSolutions/Bin/wsmAppData, где можно найти
    ReadMe(En/Ru).pdf - инструкцию по построению и исполнению приложений
    WorkSpaceManager(En/Ru).pdf - общая информация об инструменте
    wsm_help(En/Ru).chm    -  стандартный  MS Help,который также может быть вызван из приложения WorkSpaceManager.exe.

WorkSpaceManager представлен также как сочетание http-based Backend и http-based Frontend, однако:
    Многопользоательское функционарование не поддерживается.
    Не проверяется активность BackEnd.
Основная цель была - наметить дальнейшие пути построения микросервисов на базе Vip.

Application Frame

Application Frame базируется на кардинально переработанном подходе, использованном при построении WorkSpaceManager.
Application Frame  может служить как базис для создания приложений, которые:
  • имеют клиент-серверную архитектуру в форме FrontEnd и BackEnd, взаимодействующих в рамках моно-приложения или в рамках двух приложений, взаимодействующих посредством HTTP.
  • основываются на применении панели управления (ribbon)
Отличительными свойствамя являются:
  • управляющая панель  (ribbon)
    • может быть представлена в том числе как  XML структура
    • может быть легко расширена и настроена
  • позволяет расширять функции приложения путем подключения  plug-ins, построенных на базе PZL-технологии (см. VipPzlSystem )
  • использует настраиваемые словари для элементов пользовательского интерфейса, включая возможность получения мультиязычных приложений.
  • обеспечивает возможность синхронного и асинхронного обмена информацией между  FrontEnd и BackEnd.
AppFrame шаблоны помещены в директорию SpbVipTools\AppData\ProjectTemplates\Febe (копируются в Vip при выполнении SpbVipTools\Bin\SetSpbToolsToVip.exe).
Руководство AppFrameCookBook помещено в SpbVipTools\Doc\AppFrame\AppFrameCookBookEn.chm.
Директория  SpbExamples\Febe содержит работающий пример создания набора проектов.
